# Workflow: New Client Website Build

> Full website build for a new RescueMyWorkday client — from discovery audit through launch and handoff.

## Scenario

A chiropractic practice signed up for RescueMyWorkday's website + SEO package. Their current site is slow, not mobile-friendly, has no booking integration, and doesn't rank for any local keywords. They need a new site, GHL pipeline, booking flow, and basic SEO foundation.

## Agent Team

| Agent | Phase |
|-------|-------|
| Discovery Coach | Discovery & intake |
| Local SEO Specialist | SEO audit & keyword research |
| UX Architect | Site structure & wireframes |
| UI Designer | Visual design |
| Website & Landing Page Developer | Build |
| Integration & Systems Architect | GHL pipeline + booking setup |
| Automation Tester | Test automations & forms |
| Reality Checker | Final QA |
| Evidence Collector | Proof package |
| Client Support Responder | Handoff & training |

## Workflow

### Phase 0: Discovery (Week 1)
- Discovery Coach runs intake call, documents practice details
- Local SEO Specialist audits current site, GMB, citations, competitor landscape
- Output: discovery brief, SEO audit, keyword targets

### Phase 1: Strategy (Week 1)
- Head of Product defines scope and deliverables
- UX Architect creates site map and wireframes
- Senior PM sets up delivery board with milestones
- Output: approved wireframes, project plan, milestone schedule

### Phase 2: Foundation (Week 2)
- Integration & Systems Architect sets up GHL sub-account, pipeline, booking calendar
- Website Developer sets up hosting, domain, SSL
- Output: GHL ready, hosting live, domain configured

### Phase 3: Build (Weeks 2-3)
- UI Designer creates visual mockups from wireframes
- Website Developer builds pages from approved designs
- Content Creator writes service pages, about page, FAQ
- Local SEO Specialist implements schema markup and on-page SEO
- Output: complete site on staging URL

### Phase 4: QA (Week 3)
- Automation Tester tests all forms, booking flow, GHL pipeline
- Reality Checker runs full QA pass
- Evidence Collector captures proof package
- Performance Tester verifies Core Web Vitals
- Accessibility Auditor checks WCAG compliance
- Output: QA verdict, evidence package, defect list (if any)

### Phase 5: Launch (Week 4)
- Website Developer switches DNS and goes live
- Local SEO Specialist submits sitemap, verifies indexation
- Client Support Responder sends handoff email with login credentials and training guide
- Output: live site, client trained, monitoring active

### Phase 6: Operate (Ongoing)
- Analytics Reporter sends monthly performance reports
- Local SEO Specialist monitors rankings and optimizes
- Review & Reputation Manager sets up review generation
- Account Strategist conducts quarterly business review

## Quality Gates
- Phase 1→2: Approved wireframes and project plan
- Phase 3→4: All pages built and content in place on staging
- Phase 4→5: Reality Checker APPROVED verdict with evidence package
- Phase 5→6: Client confirms site is live and they can access GHL dashboard

## Total Timeline: 3-4 weeks
